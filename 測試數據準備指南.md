# 測試數據準備指南

## 一、快速測試數據生成

### 方法 1：使用後台管理介面手動建立

1. 打開管理後台
2. 點擊「建立項目」
3. 按照以下模板填寫：

#### 模板 1：完整資料案例
```
項目名稱：智慧金融風控中台
摘要概述：透過 AI 治理中台即時分析交易數據,阻詐成功率提升 400%。
亮點：採用機器學習模型，即時偵測異常交易模式
建置功能：LLM+專案人力20天
客戶名稱：某大型銀行
案例上線日期：2026/01/15
標籤：金融、經營管理、AI治理中台、AI應用
封面圖片：https://picsum.photos/id/1015/800/600
解決方案說明：採用微服務架構，整合多個數據源，運用深度學習模型進行即時風險評估。
解決方案圖片：https://picsum.photos/id/1018/800/600
```

#### 模板 2：最少資料案例（測試必填欄位）
```
項目名稱：測試案例 - 最少資料
摘要概述：這是一個只填寫必填欄位的測試案例
（其他欄位留空）
```

#### 模板 3：無圖片案例（測試問號顯示）
```
項目名稱：測試案例 - 無圖片
摘要概述：這個案例沒有圖片，應該顯示數位顆粒問號
（圖片欄位留空）
```

## 五、快速測試腳本

### 使用瀏覽器 Console 快速建立測試數據

打開瀏覽器 Console（F12），執行以下代碼：

```javascript
// 快速建立 10 個測試案例
const testCases = [
  {
    title: '測試案例 1 - 完整資料',
    description: '這是一個包含所有欄位的完整測試案例',
    highlights: '亮點：採用最新技術',
    features: '建置功能：LLM+專案人力20天',
    client: '測試客戶 A',
    launchDate: '2026-01-15',
    imageUrl: 'https://picsum.photos/id/1015/800/600',
    solutionDescription: '詳細的解決方案說明',
    solutionImageUrl: 'https://picsum.photos/id/1018/800/600',
    tagIds: ['t1', 't7', 't14'] // 金融、經營管理、AI應用
  },
  {
    title: '測試案例 2 - 無圖片',
    description: '這個案例沒有圖片，測試問號顯示',
    highlights: '',
    features: '',
    client: '',
    launchDate: '',
    imageUrl: '',
    solutionDescription: '',
    solutionImageUrl: '',
    tagIds: ['t2', 't10'] // 醫療、產品創新
  },
  // ... 可以繼續添加更多案例
];

// 儲存到 localStorage
const currentState = JSON.parse(localStorage.getItem('tpi-solution-hub-state') || '{"cases":[],"tags":[],"techDomains":[]}');
testCases.forEach(caseData => {
  const newCase = {
    ...caseData,
    id: `test-${Date.now()}-${Math.random()}`,
    dateAdded: Date.now()
  };
  currentState.cases.push(newCase);
});
localStorage.setItem('tpi-solution-hub-state', JSON.stringify(currentState));
location.reload();
```

## 七、常見問題排查

### 問題 1：案例沒有顯示
- 檢查 localStorage 是否有資料
- 檢查案例是否有必填欄位
- 檢查瀏覽器 Console 是否有錯誤

### 問題 2：圖片不顯示
- 檢查圖片 URL 是否有效
- 檢查網路連線
- 無圖片應顯示問號

### 問題 3：搜尋無結果
- 檢查搜尋關鍵字是否正確
- 檢查案例資料是否包含關鍵字
- 清除搜尋條件重試

### 問題 4：儲存失敗
- 檢查必填欄位是否填寫
- 檢查瀏覽器 Console 錯誤
- 檢查 localStorage 是否可用
