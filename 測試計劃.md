# TPI Solution Hub 測試計劃

## 測試環境準備

### 前置條件
1. 開發伺服器運行：`npm run dev`
2. 瀏覽器訪問：`http://localhost:3000`
3. 清除瀏覽器 localStorage（首次測試）

## 二、管理後台功能測試

### 2.1 案例管理

#### 測試案例 2.1.1：新增案例（完整資料）
- **步驟**：
  1. 點擊右上角「管理」按鈕
  2. 點擊「建立項目」
  3. 填寫所有欄位：
     - 項目名稱（必填）
     - 封面圖片（可選，測試上傳和 URL）
     - 摘要概述（必填）
     - 亮點（可選）
     - 解決方案圖表/圖片（可選）
     - 詳細解決方案說明（可選）
     - 客戶名稱（後台專用）
     - 案例上線日期（後台專用）
     - 建置功能（後台專用）
     - 選擇標籤
  4. 點擊「儲存記錄」
- **預期結果**：
  - 表單正確顯示
  - 所有欄位可以輸入
  - 圖片可以上傳或輸入 URL
  - 標籤可以選擇
  - 儲存後顯示成功動畫
  - 案例出現在列表中
  - 前台可以查看

#### 測試案例 2.1.2：新增案例（最少資料）
- **步驟**：
  1. 點擊「建立項目」

  4. 點擊「儲存記錄」
- **預期結果**：

## 七、測試數據準備

### 7.1 測試案例資料

建議準備以下類型的測試案例：

1. **完整資料案例**（所有欄位都有）
2. **最少資料案例**（只有必填欄位）
3. **無圖片案例**（測試問號顯示）
4. **不同產業別案例**（金融、醫療、政府等）
5. **不同業務別案例**（經營管理、產品創新等）
